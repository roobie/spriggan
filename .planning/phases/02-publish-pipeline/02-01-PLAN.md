  <action>
Create `.github/workflows/publish.yml` with exactly three jobs: `gate`, `npm`, `jsr`.

**Workflow trigger:**
```yaml
name: Publish
on:
  push:
    tags:
      - 'v*'
```

**gate job** (adapted from ci.yml pattern + version check from RESEARCH.md):
- `runs-on: ubuntu-latest`
- Step 1: `actions/checkout@v4`
- Step 2: `jdx/mise-action@v2`
- Step 3: `run: mise run check`
- Step 4: Named "Verify tag matches package versions", runs shell script:
  ```
  TAG="${GITHUB_REF_NAME#v}"
  PKG="$(node -p "require('./package.json').version")"
  JSR="$(node -p "require('./jsr.json').version")"
  if [ "$PKG" != "$TAG" ]; then echo "Version mismatch: package.json=$PKG tag=$TAG"; exit 1; fi
  if [ "$JSR" != "$TAG" ]; then echo "Version mismatch: jsr.json=$JSR tag=$TAG"; exit 1; fi
  echo "Version check passed: $TAG"
  ```

**npm job** (from slog template, simplified):
- `needs: gate`
- `runs-on: ubuntu-latest`
- `permissions:` with `contents: read` and `id-token: write`
- Step 1: `actions/checkout@v4`
- Step 2: `actions/setup-node@v4` with `node-version: '22'` and `registry-url: 'https://registry.npmjs.org'`
- Step 3: `run: npm publish --access public --provenance` with `env: NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}`
- No `npm ci` step (no build step needed -- source is published directly)

**jsr job** (from slog template, simplified -- no npm ci, no setup-node):
- `needs: gate`
- `runs-on: ubuntu-latest`
- `permissions:` with `contents: read` and `id-token: write`
- Step 1: `actions/checkout@v4`
- Step 2: `denoland/setup-deno@v2`
- Step 3: `run: deno publish`
- No `npm ci` step (Deno does not read node_modules; jsr.json is the manifest)
- No `actions/setup-node` (not needed for deno publish)

**Anti-patterns to avoid (per RESEARCH.md):**
- Do NOT put version check inside npm or jsr jobs -- it belongs in gate
- Do NOT add `npm ci` to jsr job (Spriggan has no npm: specifiers)
- Do NOT use `mise-action` in npm or jsr jobs (dead weight)
- Do NOT make jsr depend on npm or vice versa -- both depend only on gate
