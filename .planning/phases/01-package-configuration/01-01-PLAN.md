    - src/spriggan.js (full file — the source tsc will emit declarations from)
    - .gitignore (current state before adding dist/)
  </read_first>
  <action>
1. Create `tsconfig.build.json` at repo root with these exact contents:

```json
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    "noEmit": false,
    "emitDeclarationOnly": true,
    "rootDir": "./src",
    "outDir": "./dist",
    "declarationDir": "./dist"
  },
  "include": ["src/spriggan.js"]
}
```

Key overrides:
- `noEmit: false` — overrides base tsconfig's `noEmit: true` which silently blocks all output
- `emitDeclarationOnly: true` — no JS output, only .d.ts and .d.ts.map
- `include: ["src/spriggan.js"]` — MUST override base config's `["src/**/*", "examples/**/*"]` because examples/ has type errors that abort emit
- `rootDir/outDir/declarationDir` all point to dist/ for clean separation

2. Add `dist/` to `.gitignore` (append on new line after existing `node_modules/`).

3. Run the declaration emit: `rm -rf dist && bunx tsc -p tsconfig.build.json`

4. Run the TYPE-01 comparison: `diff src/spriggan.d.ts dist/spriggan.d.ts`

5. Evaluate the diff. Key things to check per RESEARCH.md Pitfall 5:
   - Does tsc emit `app` correctly as part of `createSpriggan`'s return type (not as a top-level function)?
   - Does tsc emit the generic `AppConfig<T, M>` and `AppApi<T, M>` shapes?
   - Are the 5 built-in effect union types present?
   - Is the overall quality comparable to the handwritten version?

6. Make the declaration approach decision:
   - If tsc output is good enough: use tsc-emitted (dist/ path). Record in SUMMARY.
   - If handwritten is better: keep handwritten in src/. Record in SUMMARY. Copy handwritten to dist/ or adjust exports to point at src/.

7. Record the decision clearly — Plan 02 needs to know whether exports.types points at `./dist/spriggan.d.ts` or `./src/spriggan.d.ts`.
