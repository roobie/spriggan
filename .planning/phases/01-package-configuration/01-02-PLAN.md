Critical rules:
- `"jsr.json"` MUST be the first item in `publish.include` — JSR requires the manifest itself be part of the publish set when using an include list. Omitting it causes a confusing error (PITFALLS.md Pitfall 4).
- `exports` points directly to `./src/spriggan.js` (not a conditions object like npm — JSR resolves types via @ts-self-types directive in the .js file)
- Version MUST match package.json version (1.0.0) — CI will enforce this
- No `publish.exclude` needed (the include list is already a whitelist)
